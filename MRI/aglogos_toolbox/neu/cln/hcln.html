HCLN - Help for cleaning inteferance noise
    

    Cleaning of interference noise from neural signals is based on PCA.  Image sesgmentation
    and slicing introduces unique noise pattern in term of timing, then PCA will be applied
    segment/slice by segment/slice.    For example, 2 segmetns, 2 slices corresponds to 4
    unique patterns and 4 PCA will be applied.  Gradient noise pattern must be recorded into
    ADF/ADFW file and ParaVision must send every trigger signal for each MRI event to QNX.
    SESCLNADJEVT detects precise timings of MRI event.
    SESGETCLN runs cleaning functions CLNMAIN and CLNADF.
    

    Control flags/setting for cleaning is, (as of 19.08.05)
          ANAP.DEBUG = 0;              % Debug flag for clnadf, not clnmain
          ANAP.DECFRAC = 3;              % Decimation factor
          ANAP.HIGHPASS = 1;              % Cutoff freq for high pass (in Hz)
          ANAP.PREPCA = 0.0015;    % Pre-period for PCA data in sec.
          ANAP.PCALAGS = 5;              % Lags for selecting PC's to remove
          ANAP.PCACOEF = 0.25;        % correlation coefficient (works fine)
          ANAP.NOPCS = 6;              % Number of singular values to compute
          ANAP.NOREM = 6;              % Number of PCs to remove
          ANAP.PCAGAP = 1;              % Removes gaps around edges of PCA data.
          ANAP.OUTLIERS = 1;              % Set if check for outliers is desired
          ANAP.SAVE = 1;              % Create/Append MAT file
          ANAP.SAVEAS_ADX = 0;              % Save decimated data into a separate data.
          ANAP.SAVEGRA = 0;              % Save gradient noise


    TIPS :
        * Cleaning parameters can be set in the description file as ANAP.clnpar.xxxx.
        * If memory problem, run Matlab without java virtual machine (-nojvm option).
            Since cleaning function uses decimate() that allocates 2 more memory of data,
            make sure that you have enough memory (2G) when data is large (~30min collected data).

    TROUBLE SHOOT :
    - memory problem
        * Run the Matlab without java virtal machine (-nojvm option).
        * Make sure enough memory (2GB) when data is large (~30min collected data).
    - memory problem, STILL
        * DO NOT RUN ANY OTHER PROGRAMS (OUTLOOK,IE,NETSCAPE,etc) OTHER THAN MATLAB.
        * Next time, collect data with shorter duration.
        * Basically cleaning including decimation of 10min data will require...
                21000(Hz)*60(sec)*10(min)*8(bytes)*3/1024/1024 = 288.4MB
            and 1/3 of 288.4MB must be allocated in CONTINUOUS SPACE.
        * You can test like following
                >> DATA_minutes = 30;
                >> rawdata = rand(21000*60*DATA_minutes,1);
                >> decdata = decimate(rawdata,3);
    - NOISE REMAINED
        * Set ANAP.clnpar.PCACOEF in the session file with lower value like 0.15 and run again.
        * Make sure that you have enough data for PCA, interferance pattern is unique in each
            segment/slice.  Therefore, if the pattern doen't appear many times in data, PCA will
            not detect meaningful noise componets.
    


    EXAMPLE :
        >> sesdumppar(SESSION,ExpNo);            % saves experiment parameters as matlab files
        >> sesclnadjevt(SESSION,ExpNo);        % detects precise timings of MRI event
        >> sesgetcln(SESSION,ExpNo);              % cleans noise
