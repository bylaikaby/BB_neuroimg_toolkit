HGETSTARTED -- Help to Get Started (22.Nov.2007)

This help tells how to get started with AGLOGO analysis package.
Analsys flow can be like   
  1. Raw data preparation (data conversion, copy of raw data to user's disk etc)
  2. Preparing a description file
  3. Loading experiment information (DGZ, acqp/reco, stm etc.)
  4. Loading MRI data (2dseq)
  5. Loading neural signals (ADF/ADFW) into matlab format.
  6. Preprocessing MRI data (roiTs/troiTs)
  7. Preprocessing neural data (ClnSpc, blp Spkt etc.)
  8. Statistical analysis (corr./glm analysis)
  9. Displaying results (mview(), nview()).
For typical MRI session, the program shuold be run like
>> SES = 'XyzAbc';
>> sesdumppar(SES);      % loading experiment information
>> sesascan(SES);        % loading anatomy scans
>> sesimgload(SES);      % loading 2dseq as matlab format as tcImg
>> mroi(SE);  pause;     % define ROIs
>> sesareats(SES);       % preprocessing MRI time courses for ROIs as roiTs
>> sesgettrial(SES);     % split roiTs into trials if required,  troiTs
>> sesgrpmake(SES);      % average signals over the group
>> sesgroupcor(SES);     % correlation analysis to roiTs/troiTs
>> sesgroupglm(SES);     % glm analysis to roiTs/troiTs
>> sesgetmask(SES);      % create some masking data if required
>> mview(SES,GrpName);   % see results
If neural data,
>> sesclnadjevt(SES);    % detect timings of gradient noises (only for MRI)
>> sesgetcln(SES);       % loading ADF/ADFW as Cln with denoising/decimation
>> sesgetblp(SES);       % extract bands from Cln as blp
>> sesgetspk(SES);       % extract spikes from Cln as Spkt/Sdf
>> sesgrpmake(SES);      % average signals over the group
>> nview(SES,GrpName);   % see results

Raw data preparation

Many types of data file will be created after your data acquisition.  THERE IS NO PERSON TO TAKE CARE OF DATAFILES OTHER THAN YOURSELF.  Those files may be deleted by someone else because of disk space problem to recover from computer crash. It is very very important to copy/move datafile after your experiment.
According to data collection you made, following datafiles will be created.
    .evt(dgz),.emd,.spk,.adf  : event timings etc.  :  /usr/data/ in QNX
    .adfw                     : analog waveforms    :  Windows running WinStreamer
    .stm,.pdm,.hst,.rfp       : stimulus parameters :  Windows running Stim/Stimcon
Target directory of raw data must be organized as
.dgz .adfw must be stored in "/xxxx/SESSION_NAME".
.stm/pdm/hst/rfp must be stored in "/xxxx/SESSION_NAME/stmfiles".
 MRI raw data must be sored in "yyyy/SESSION_NAME".
Note that dgz/adfw/stm etc can be in the same session directory as MRI data.  MRI raw data (2dseq, acqp/reco etc) can be found usually in "//wks8/guest/SESSION_NAME" (4.7T) or "//wks6/guest/SESSION_NAME" (7T).

[TO DO IN QNX]
1.    Run a shell
2.    Mount or Move to your own data directory, NOT INSIDE QNX
3.    Run convdata to compile evt/emd/spk into dgz file.
4.    Move all relevant file to your directory.

Example)
[d2n3:lab] cd /usr/lab/projects/YOUR_DIR             % move to your own directory
[d2n3:YOUR_DIR] convdata filename                    % generate .dgz in YOUR_DIR
[d2n3:YOUR_DIR] mv /usr/data/evtdat/YOURFILE.*  .    % moving .evt to YOUR_DIR
[d2n3:YOUR_DIR] mv /usr/data/spkdat/YOURFILE.*  .    % moving .spk to YOUR_DIR
[d2n3:YOUR_DIR] mv /usr/data/emdat/YOURFILE.*  .     % moving .emd to YOUR_DIR

[TO DO IN WINSTREAMER]
1.    Run Korn-shell (MKS toolkit)
2.    Go to the directory where ADFW stored.
3.    Run convadfw to convert raw ADFW to processed ADFW.
4.    Move ADFW into your own directory.

Due to machine performance, data collection, and conversion (reading/writing/copying whatever causes disk access) must be done exclusively, otherwise heavy disk access destroys data collection.

Example)
[F:/adfdat] mkdir cvt                               % make sure cvt directory exists
[F:/adfdat] convadfw YOURFILE.*                     % convert ADFW
[F:/adfdat] mv cvt/YOURFILE.*  YOUR_DRV/YOUR_DIR    % move converted ADFW
[F:/adfdat] mv YOURFILE.*  YOUR_DRV/YOUR_DIR/raw    % move the originals, if need


[TO DO IN STIM COMPUTER]
1.    Go to MriStim/params directory.
2.    Move .stm/.pdm/.hst/.rtp/.prt to your own data direcoty.
3.    If receptive field file (.rfp) exists, then move it too (it is usually in MriStim/params/rfpfiles).

Note that MriSim/params directory may differ from setup to setup.  For example, it is D:\Mri\MriStim\params for win26 (as of 5/30/2005).

Preparing a session describing file

To analyze data, the AGLOGO package requires a file describing about the experiment session.  Each session should have its own description file.  A session is named like animal nubers and experiment identification characters, usually like M02lx1 where M02 as animal number, lx1 as experiment ID.  A template for description file is found  as "ana/@SessionTemplate.m" and this can be copied as SESSION_NAME.m to modify.  "ana" directory also has lots of examples.  For grammer tesing, sescheck() or sescheckLv2() can be used.

Session files are suppposed to have direcory settings, analysis parameters, data filenames and so on.
"SYSP" sets data direcories for adf/adfw and 2dseq.
"ANAP" sets analysis parameters for the session.
"ASCAN" sets information about anatomical scans.
"CSCAN" sets information about basic epi13 scans.
"GRPP"  sets default parameters for all groups.  If each group has the same parameter(s), then programs will overwrite values by "GRPP" and use values by group's own.
"GRP.xxx" sets experimet information for each group (repetition of the same data collection, experiment).
"EXPP" provides filename, scan/reco number for each data collection.


Here is an example for the session ABC.def
%ABCdef - Experiment Aim here
% SESSION : AGCdef in the MR-Room
% EXPDATE : dd.mm.yy
% NOTES: whatever...

%=======================================================================
% basic information : data directories, session quality
%=======================================================================
SYSP.DataNeuro = '//Win49/M/DataNeuro/';  % dir. for adf/adfw/dgz
SYSP.DataMri = '//Wks8/guest/nmr/';     % dir. for 2dseq
SYSP.dirname = 'ABC.def';
ANAP.Quality = -1; % Percent (all exps good activation)
 
%=======================================================================
% CATEGORIES (CTG) OF  EXPS/GROUPS/SIGS, if exist or needed
%=======================================================================
% CTG.exclGrps = ...
% CTG.inclGrps = ...
% CTG.rfGrps{X} = ...
% CTG.chcfGrps{X} = ...
% CTG.winGrps{X} = ...
% CTG.ImgGrps{X} = ...
% CTG.ImgSpoGrps = ...
 
%=======================================================================
% default analysis parameters : see also getses.m/getanap.m
% this must be overwritten in each group, if different.
%=======================================================================
% ANAP.xxxx = ...
 
%=======================================================================
% MOVIE analysis (if exist) : THIS MUST BE SET CAREFULLY.
%=======================================================================
ANAP.revcor.Frame  = 1;  % For display purposes
ANAP.revcor.TOFFSET  = [0];
ANAP.revcor.LFP_THR  = [3];
ANAP.revcor.MUA_THR  = [3];
ANAP.revcor.BadRFChan = [];  % For display purposes
ANAP.revcor.MovPos  = [-2.2 -3.6 20 15];  % [centx centy width height]
%ANAP.revcor.NO_AVG  = 2000;  % see also getrf.m
 
 
%=======================================================================
% COHERENCE / CONTRAST analysis (if exist) : THIS MUST BE SET CAREFULLY.
%=======================================================================
ANAP.confunc.maxchan = 16;
ANAP.confunc.eledist = 2; % 2mm
ANAP.confunc.eleconfig = [ 01 02 03 04; ...
         05 06 07 08; ...
         09 10 11 12; ...
         13 14 15 16];
 
%=======================================================================
% ROI for MRI experiments, if exist or needed, then put here
% brain-ROI must be defined
% test-ROI should be defined, because if exists it's used to test
% scan-stability
% All roi should be with lower case
% model roi is the one we may use for xcor analysis
%=======================================================================
ROI.groups = {'all'};                              % SuperAvg (see HROI)
ROI.names = {'brain'; 'ele'; 'v1'; 'v2'; 'test'}; % Define desired ROIs
ROI.model = 'ele';                                % Group to use as model
 
 
 
%=======================================================================
% ANATOMY scans (if exist gefi/mdeft/ir/msme)
% PLEASE MAKE SURE THE ELECTRODE POSITION AND SLICE IS 100% CORRECT
%=======================================================================
ASCAN.gefi{1}.info  = 'Electrode localization scan';
ASCAN.gefi{1}.scanreco = [1 1];
ASCAN.gefi{1}.imgcrop = [128 80 136 88];
 
ASCAN.mdeft{1}.info  = 'Anatomy mdeft';
ASCAN.mdeft{1}.scanreco = [2 1];
ASCAN.mdeft{1}.imgcrop = [128 80 136 88];
 
ASCAN.mdeft{2}.info  = 'Anatomy mdeft';
ASCAN.mdeft{2}.scanreco = [12 1];
ASCAN.mdeft{2}.imgcrop = [128 80 136 88];
 
ASCAN.ir{1}.info  = 'Anatomy ir';
ASCAN.ir{1}.scanreco = [5 1];
ASCAN.ir{1}.imgcrop  = [128 80 136 88];
 
%=======================================================================
% EPI13 : basic functional scans (if exist)
%=======================================================================
CSCAN.epi13{1}.info  = 'Polars Stim';
CSCAN.epi13{1}.ana  = {};
CSCAN.epi13{1}.scanreco = [7 1];
CSCAN.epi13{1}.imgcrop = [32 20 34 22];
CSCAN.epi13{1}.v  = {[1 0 1 0 1 0 1 0]};
CSCAN.epi13{1}.t  = {[8 8 8 8 8 8 8 8]};
 
 
%=======================================================================
% default group parameters
% this must be overwritten in each group, if different.
%=======================================================================
GRPP.daqver  = 2.00;  % DAQ program version: 2=nl+ym; 1=nl;
GRPP.imgcrop = [56 10 56 36]; % x, y, width, height
GRPP.ana  = {'gefi'; 1; [11 14]; [14 14]};
GRPP.hwinfo  = '';  % hardware info
GRPP.hardch  = [1 2]; % electrode numbers for ADF_CHANNELs
GRPP.softch  = [];  % invalidated channels for analysis
GRPP.grproi  = 'RoiDef'; % the name of a Group's ROI, RoiDef as default
 
%=======================================================================
% experiment groups
%=======================================================================
% test1 : test scan with 'polar'
GRP.test1.exps  = [1];  % experiment numbers
GRP.test1.expinfo = {'imaging','recording'};
GRP.test1.stminfo = 'polar test';
 
% test2 : test scan with 'polar', no event data (no-dgz)
GRP.test2.exps  = [10];  % experiment numbers
GRP.test2.expinfo = {'imaging','alert'};
GRP.test2.stminfo = 'Polar Test';
% if no-dgz or to ignore dgz,
% then should have these v/t/stmtypes values.
GRP.test2.v   = {[1 0 1 0 1 0 1 0]};  % stimulus, 0 as blank
GRP.test2.t   = {[8 8 8 8 8 8 8 8]};  % duration in volumes
GRP.test2.stmtypes = {'blank','polar'};    % 0=blank, 1=polar
 
% movie1 : 5 min movie of xxxx.avi
GRP.movie1.exps  = [2:9 11];  % experiment numbers
GRP.movie1.expinfo = {'imaging','recording'};
GRP.movie1.stminfo = 'movie1';
 
% movie2 : 5 min movie of yyyy.avi
GRP.movie2.exps  = [12 15];  % experiment numbers
GRP.movie2.expinfo = {'imaging','recording'};
GRP.movie2.stminfo = 'movie2';
 
 
%=======================================================================
% individual files (must cover all 'exps'.)
%=======================================================================
for N = 1:15,
  EXPP(N).physfile  = sprintf('abcdef_%03d.adfw',N);    % adf/adfw
  EXPP(N).videofile = sprintf('abcdef_%03d_2.adfw',N);  % for movie
  EXPP(N).scanreco  = [N+8, 1];                         % 2dseq
end

Basic functions for MRI analysis

exppar/ sesdumppar (SES,GRP) Get/Load all experimental parameteters of the session.
mri/ sesascan (SES) Load all anatomy files
mri/ sescscan (SES) Load and analyze control scans
mri/ sesimgload (SES,GRP) Load Paravision 2dseq imagefiles of a Session
mri/ mroi (SES,GRP) Utility to define Regions of Interests (ROIs)
mri/ sesareats (SES,GRP) Generate Time-Series for each area defined in ROI.names
exppar/ sesgettrial (SES,GRP) Split observation period in trials and return average response per trial.
mri/ sescorana (SES,GRP) Correlation analysis w/ simple xcor
glm/ sesglmana (SES,GRP) Performs GLM for a whole group or a set of experiments.
procs/ sesgrpmake (SES,GRP,SIGNAME) group data in the same group
mri/ sesgroupcor (SES,GRP) Correlation analysis on all group files
mri/ sesgroupglm (SES,GRP) GLM analysis on group files
mri/ sesgetmask (SES,GRP) Creates mask on the basis of p/r values of reference group to select time-series
mri/ mview (SES,GRP) Displays roiTs/troiTs

Basic functions for Neural data analysis

exppar/ sesdumppar (SES,GRP) Get/Load all experimental parameteters of the session.
cln/ sesclnadjevt (SES,GRP) Creates the ClnAdjEvt.mat file with corrected MRI events
cln/ sesgetcln (SES,GRP) Read ADF files, eliminate Grad.Noise and/or Decimate
neu/ sesclnspc (SES,GRP) Make spectrograms for the Cln signal of each experiment
neu/ sesgetblp (SES,GRP) Separate the Cln signal into freqeuncy bands for entire session
neu/ sesgetspk (SES,GRP) Extracts spikes and SDFs from Cln.dat
exppar/ sesgettrial (SES,GRP) Split observation periods into trials
procs/ sesgrpmake (SES,GRP,SIGNAME) group data in the same group
neu/ nview (SES,GRP) Displays neural signals
neu/ spkview (SES,GRP) Displays spike signals
